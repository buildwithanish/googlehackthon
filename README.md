## 💎 Core Features & How They Work

### 1. **Automated Bias Discovery Engine**
*   **How it works**: The engine uses mathematical kernels to calculate Statistical Parity and Disparate Impact Ratio. When a user uploads a CSV, FairAI profiles the data types and automatically identifies sensitive groups (Gender, Race, etc.) to perform a multi-dimensional fairness audit.
*   **Metric Insight**: We utilize the **EEOC 80% Rule** to determine if a model is legally compliant in its current state.

### 2. **Gemini 1.5 Pro Neuro-Explainer**
*   **How it works**: Raw metrics are hard to understand. We feed the fairness results into **Google Gemini 1.5 Pro** along with the dataset schema. The AI generates a human-readable "Narrative of Bias," explaining exactly *where* the model is failing and why.

### 3. **AI-Driven Remediation Roadmap**
*   **How it works**: FairAI doesn't just find problems; it fixes them. Based on the bias type (Pre-processing, In-processing, or Post-processing), the system suggests specific ML algorithms like **Reweighing (AIF360)** or **Threshold Optimization (Fairlearn)** to mitigate the bias.

### 4. **Enterprise-Grade Audit Reporting**
*   **How it works**: One-click generation of **PDF**, **Word**, and **Simulation PowerPoint** reports. These are not simple text files—they are structured documents with cover pages, data visualizations, and legal-style summaries suitable for board-level reviews.

### 5. **Hybrid Cloud Connectivity**
*   **How it works**: Our **Vertex AI Simulation Failover** ensures that even if a backend API is unreachable, the system uses cloud-simulated intelligence to provide a seamless user experience, making it "Always-On" for mission-critical auditing.
